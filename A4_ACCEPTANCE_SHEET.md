## 二、完成的基本功能

| # | 功能 | 说明 |
|---|------|------|
| 1 | 拦截功能 | 本地文件中 IP=0.0.0.0 的域名返回 NXDOMAIN（RCODE=3），不返回 0.0.0.0 |
| 2 | 本地查询 | 本地文件中有效 IP 地址直接返回 A 记录 |
| 3 | 中继功能 | 本地文件中没有的域名，转发到外部 DNS 服务器查询 |
| 4 | 中继结果本地存储 | 上游响应中的 Answer 记录写入内存 CacheTable（哈希表），TTL 过期自动清理 |
| 5 | 多客户端支持 | 单线程 select() 事件循环 + DNS ID 转换（TransactionTable），支持并发 |
| 6 | 命令行输出 | 序号 + 时间戳 + 客户端 IP + 查询类型 + 域名 + 查询情况 + 查询结果 |

## 三、完成的增强功能

| # | 功能 | 说明 |
|---|------|------|
| 1 | 一个域名多 IP | 支持 ANCOUNT > 1，如 multi.example.com → 3 个 A 记录 |
| 2 | AAAA 记录 | IPv6 地址解析 |
| 3 | CNAME 记录 | 别名解析 |
| 4 | NS 记录 | 名称服务器记录 |
| 5 | PTR 记录 | 反向指针记录（in-addr.arpa） |
| 6 | MX 记录 | 邮件交换记录（含优先级） |
| 7 | 动态 Cache | 哈希表存储上游响应，TTL 过期自动清理 |
| 8 | DNS over TCP | 上游 DNS 查询支持 TCP 传输（2 字节长度前缀） |
| 9 | TC fallback | UDP 响应 TC=1 时自动通过 TCP 重新查询完整响应 |
| 10 | 超时处理 | 事务 3 秒超时清理，不可达上游时 timeout 日志 + 表回收 |

## 四、程序强项

1. **模块化设计** — 12 个模块（config / logger / platform_win / udp_socket / dns_name / dns_packet / resource_record / resource_store / cache / transaction / tcp_dns_client / main），每个模块隐藏内部实现，只暴露稳定接口
2. **安全字节读写** — 所有 DNS 报文字段使用 `read_u16_be` / `write_u16_be` 等安全函数，无强制指针转换，避免非对齐访问
3. **压缩域名解析** — 支持 DNS 压缩指针，含循环检测（最多 40 跳）
4. **DNS ID 转换** — relay_id 显式保证 ≠ client_id，Wireshark 可清晰验证
5. **开放寻址哈希 Cache** — djb2 哈希 + 线性探测，O(1) 平均查找
6. **TCP fallback** — 三层安全回退（TCP 成功 → 使用 TCP；TCP ID 不匹配 → 使用 UDP；TCP 失败 → 使用 UDP）
7. **程序输出符合验收要求** — 自然数序号 + 精确到秒的时间戳 + 客户端 IP + 查询信息 + 查询情况 + 查询结果

## 五、AI 使用说明

> 按实际情况修改。以下为建议表述：

- 部分代码框架由 AI 辅助生成（如模块骨架、构建脚本、文档初稿）
- 小组成员对以下核心实现进行了理解、调试和验证：
  - DNS 报文解析与构造（Header / Question / Answer / 压缩域名）
  - DNS ID 转换与 TransactionTable
  - 资源记录文件解析
  - Cache 哈希表与 TTL 过期
  - DNS over TCP 与 TC fallback
  - select() 并发模型
- 所有功能均经过 Wireshark 抓包验证，小组成员能解释各字段含义
